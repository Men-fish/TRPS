# Веб-агрегатор спортивных тренеров

Backend приложение на Flask + SQLite для платформы поиска и записи к спортивным тренерам.

## Технологии

- Python 3.8+
- Flask 3.0.0
- SQLite
- Flask-WTF 1.2.1 (CSRF защита)
- Werkzeug 3.0.1 (хеширование паролей)

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Запустите приложение:
```bash
python app.py
```

Приложение будет доступно по адресу: `http://localhost:5000`

3. (Опционально) Добавьте тестовых тренеров:
```bash
python add_test_trainers.py
```

## API Endpoints

### Аутентификация

#### `GET /api/csrf-token`
Получение CSRF токена для текущей сессии.

**Параметры:** нет

**Ответ (200):**
```json
{
  "csrf_token": "..."
}
```

#### `POST /auth/register`
Регистрация нового пользователя.

**Параметры (Body JSON):**
- `full_name` (string, обязательный) - ФИО пользователя (2-200 символов)
- `email` (string, обязательный) - Email адрес (уникальный, валидный формат)
- `password` (string, обязательный) - Пароль (минимум 6 символов, максимум 128)
- `passport_data` (string, обязательный) - Паспортные данные (минимум 5 символов, максимум 100)

**Ответ (201):**
```json
{
  "message": "Пользователь успешно зарегистрирован",
  "user": {
    "id": 1,
    "full_name": "Иванов Иван Иванович",
    "email": "ivan@example.com",
    "created_at": "2025-12-07 16:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 409 (дубликат email)

#### `POST /auth/login`
Авторизация пользователя. Возвращает CSRF токен.

**Параметры (Body JSON):**
- `email` (string, обязательный) - Email адрес
- `password` (string, обязательный) - Пароль

**Ответ (200):**
```json
{
  "message": "Успешная авторизация",
  "user": {
    "id": 1,
    "full_name": "Иванов Иван Иванович",
    "email": "ivan@example.com"
  },
  "csrf_token": "..."
}
```

**Ошибки:** 400 (валидация), 401 (неверные учетные данные)

#### `GET /auth/me`
Информация о текущем авторизованном пользователе.

**Параметры:** нет (требует авторизацию)

**Ответ (200):**
```json
{
  "user": {
    "id": 1,
    "full_name": "Иванов Иван Иванович",
    "email": "ivan@example.com",
    "created_at": "2025-12-07 16:00:00"
  }
}
```

**Ошибки:** 401 (не авторизован)

#### `POST /auth/logout`
Выход из системы.

**Параметры:** нет (требует авторизацию)

**Ответ (200):**
```json
{
  "message": "Успешный выход из системы"
}
```

**Ошибки:** 401 (не авторизован)

### Тренеры

#### `GET /trainers`
Список тренеров с фильтрацией и поиском.

**Query параметры:**
- `specialization` (string, опциональный) - Фильтрация по специализации (например: `?specialization=Фитнес`)
- `search` (string, опциональный) - Поиск по ФИО (например: `?search=Иванов`)

**Ответ (200):**
```json
{
  "trainers": [
    {
      "id": 1,
      "full_name": "Иванов Иван Иванович",
      "specialization": "Фитнес",
      "description": "Опытный фитнес-тренер...",
      "created_at": "2025-12-07 13:00:00"
    }
  ],
  "count": 1
}
```

#### `GET /trainers/<id>`
Детальная информация о тренере с отзывами и средней оценкой.

**Параметры:**
- `id` (int, path) - ID тренера

**Ответ (200):**
```json
{
  "trainer": {
    "id": 1,
    "full_name": "Иванов Иван Иванович",
    "specialization": "Фитнес",
    "description": "Опытный фитнес-тренер...",
    "created_at": "2025-12-07 13:00:00",
    "rating": {
      "average": 4.5,
      "count": 10
    }
  },
  "reviews": [
    {
      "id": 1,
      "user_name": "Петров Петр",
      "text": "Отличный тренер!",
      "rating": 5,
      "created_at": "2025-12-07 14:00:00",
      "updated_at": null
    }
  ]
}
```

**Ошибки:** 404 (тренер не найден)

### Записи на тренировки

#### `GET /records`
История записей текущего авторизованного пользователя.

**Query параметры:**
- `status` (string, опциональный) - Фильтрация по статусу: `pending`, `confirmed`, `cancelled`, `completed` (например: `?status=pending`)

**Параметры:** нет (требует авторизацию)

**Ответ (200):**
```json
{
  "records": [
    {
      "id": 1,
      "trainer": {
        "id": 1,
        "full_name": "Иванов Иван Иванович",
        "specialization": "Фитнес"
      },
      "datetime": "2025-12-10 14:00:00",
      "status": "pending",
      "created_at": "2025-12-07 15:00:00"
    }
  ],
  "count": 1
}
```

**Ошибки:** 401 (не авторизован)

#### `POST /records`
Создание новой записи на тренировку.

**Параметры (Body JSON):**
- `trainer_id` (int, обязательный) - ID тренера
- `datetime` (string, обязательный) - Дата и время тренировки в формате `YYYY-MM-DD HH:MM:SS` (должна быть в будущем)

**Требует:** авторизацию + CSRF токен (заголовок `X-CSRF-Token`)

**Ответ (201):**
```json
{
  "message": "Запись успешно создана",
  "record": {
    "id": 1,
    "trainer": {
      "id": 1,
      "full_name": "Иванов Иван Иванович",
      "specialization": "Фитнес"
    },
    "datetime": "2025-12-10 14:00:00",
    "status": "pending",
    "created_at": "2025-12-07 15:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 401 (не авторизован), 403 (CSRF), 404 (тренер не найден)

#### `PUT /records/<id>`
Изменение существующей записи на тренировку.

**Параметры:**
- `id` (int, path) - ID записи

**Параметры (Body JSON, опциональные):**
- `datetime` (string) - Новая дата и время в формате `YYYY-MM-DD HH:MM:SS` (должна быть в будущем)
- `status` (string) - Новый статус: `pending`, `confirmed`, `cancelled`, `completed`

**Требует:** авторизацию + CSRF токен (заголовок `X-CSRF-Token`)

**Ограничения:** можно изменять только свои записи

**Ответ (200):**
```json
{
  "message": "Запись успешно обновлена",
  "record": {
    "id": 1,
    "trainer": {...},
    "datetime": "2025-12-10 15:00:00",
    "status": "confirmed",
    "created_at": "2025-12-07 15:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 401 (не авторизован), 403 (CSRF или нет прав), 404 (запись не найдена)

#### `DELETE /records/<id>`
Отмена записи на тренировку (изменение статуса на `cancelled`).

**Параметры:**
- `id` (int, path) - ID записи

**Требует:** авторизацию + CSRF токен (заголовок `X-CSRF-Token`)

**Ограничения:** можно отменять только свои записи

**Ответ (200):**
```json
{
  "message": "Запись успешно отменена",
  "record": {
    "id": 1,
    "status": "cancelled"
  }
}
```

**Ошибки:** 401 (не авторизован), 403 (CSRF или нет прав), 404 (запись не найдена)

### Отзывы

#### `GET /reviews/<trainer_id>`
Список отзывов о тренере.

**Параметры:**
- `trainer_id` (int, path) - ID тренера

**Ответ (200):**
```json
{
  "trainer": {
    "id": 1,
    "full_name": "Иванов Иван Иванович",
    "specialization": "Фитнес"
  },
  "reviews": [
    {
      "id": 1,
      "user_name": "Петров Петр",
      "text": "Отличный тренер!",
      "rating": 5,
      "created_at": "2025-12-07 14:00:00",
      "updated_at": null
    }
  ],
  "count": 1
}
```

**Ошибки:** 404 (тренер не найден)

#### `POST /reviews`
Создание нового отзыва о тренере.

**Параметры (Body JSON):**
- `trainer_id` (int, обязательный) - ID тренера
- `text` (string, обязательный) - Текст отзыва (не пустой, максимум 2000 символов)
- `rating` (int, обязательный) - Оценка от 1 до 5

**Требует:** авторизацию + CSRF токен (заголовок `X-CSRF-Token`)

**Ограничения:** один пользователь может оставить только один отзыв на тренера

**Ответ (201):**
```json
{
  "message": "Отзыв успешно создан",
  "review": {
    "id": 1,
    "user_name": "Иванов Иван Иванович",
    "trainer_id": 1,
    "text": "Отличный тренер!",
    "rating": 5,
    "created_at": "2025-12-07 15:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 401 (не авторизован), 403 (CSRF), 404 (тренер не найден), 409 (дубликат отзыва)

#### `PUT /reviews/<id>`
Редактирование существующего отзыва.

**Параметры:**
- `id` (int, path) - ID отзыва

**Параметры (Body JSON, опциональные):**
- `text` (string) - Новый текст отзыва (не пустой, максимум 2000 символов)
- `rating` (int) - Новая оценка от 1 до 5

**Требует:** авторизацию + CSRF токен (заголовок `X-CSRF-Token`)

**Ограничения:** можно редактировать только свои отзывы

**Ответ (200):**
```json
{
  "message": "Отзыв успешно обновлен",
  "review": {
    "id": 1,
    "user_name": "Иванов Иван Иванович",
    "trainer_id": 1,
    "trainer_name": "Иванов Иван Иванович",
    "text": "Обновленный текст отзыва",
    "rating": 4,
    "created_at": "2025-12-07 15:00:00",
    "updated_at": "2025-12-07 16:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 401 (не авторизован), 403 (CSRF или нет прав), 404 (отзыв не найден)

### Расписание работы тренеров

#### `GET /schedules/trainer/<trainer_id>`
Получить расписание работы тренера.

**Параметры:**
- `trainer_id` (int, path) - ID тренера

**Ответ (200):**
```json
{
  "trainer": {
    "id": 1,
    "full_name": "Иванов Иван Иванович",
    "specialization": "Фитнес"
  },
  "schedules": [
    {
      "id": 1,
      "day_of_week": 0,
      "day_name": "Понедельник",
      "start_time": "09:00",
      "end_time": "18:00",
      "is_active": true,
      "created_at": "2025-12-07 16:00:00",
      "updated_at": null
    }
  ],
  "count": 1
}
```

**Ошибки:** 404 (тренер не найден)

#### `POST /schedules`
Создание нового расписания для тренера.

**Параметры (Body JSON):**
- `trainer_id` (int, обязательный) - ID тренера
- `day_of_week` (int, обязательный) - День недели: 0=понедельник, 1=вторник, ..., 6=воскресенье
- `start_time` (string, обязательный) - Время начала работы в формате `HH:MM` (например, `09:00`)
- `end_time` (string, обязательный) - Время окончания работы в формате `HH:MM` (например, `18:00`)
- `is_active` (boolean, опциональный) - Активно ли расписание (по умолчанию `true`)

**Ответ (201):**
```json
{
  "message": "Расписание успешно создано",
  "schedule": {
    "id": 1,
    "trainer_id": 1,
    "day_of_week": 0,
    "day_name": "Понедельник",
    "start_time": "09:00",
    "end_time": "18:00",
    "is_active": true,
    "created_at": "2025-12-07 16:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 404 (тренер не найден), 409 (расписание на этот день уже существует)

#### `PUT /schedules/<id>`
Обновление существующего расписания.

**Параметры:**
- `id` (int, path) - ID расписания

**Параметры (Body JSON, опциональные):**
- `start_time` (string) - Новое время начала в формате `HH:MM`
- `end_time` (string) - Новое время окончания в формате `HH:MM`
- `is_active` (boolean) - Новый статус активности

**Ответ (200):**
```json
{
  "message": "Расписание успешно обновлено",
  "schedule": {
    "id": 1,
    "trainer_id": 1,
    "day_of_week": 0,
    "day_name": "Понедельник",
    "start_time": "10:00",
    "end_time": "19:00",
    "is_active": true,
    "created_at": "2025-12-07 16:00:00",
    "updated_at": "2025-12-07 17:00:00"
  }
}
```

**Ошибки:** 400 (валидация), 404 (расписание не найдено)

#### `DELETE /schedules/<id>`
Удаление расписания.

**Параметры:**
